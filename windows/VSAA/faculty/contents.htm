2.3. Installing Internet Information Services (IIS)

Internet Information Services (IIS) must be installed prior to installing Visual Studio .NET and your Assignment Manager server, or Assignment Manager will not work. IIS is not installed by default in the installation of Windows .NET Standard Server, Windows .NET Enterprise Server, Windows 2000 Professional, and Windows XP Professional. If you are installing Assignment Manager on one of these operating systems, use the following procedure to install IIS.

To install Internet Information Service (IIS)

  1. Click the Start button, point to Settings, and then click Control Panel.
  2. Double-click Add/Remove Programs.
  3. Click Add/Remove Windows Components.
  4. Select Internet Information Services (IIS).
  5. Click Next.
  6. When the component installation is complete, the Windows Component Wizard dialog box appears. Click Finish to complete the installation.

If you are using Windows .NET Server, you must enable ASP.NET and change some settings.

To enable IIS on Windows .NET Server

  1. Click the Start button, point to Settings, and then click Control Panel.
  2. Choose Administrative Tools and then select Internet Information Services.
  3. In the left pane, right-click on your machine name and choose Security.
  4. In the IIS Security Lockdown Wizard, click Next.
  5. On the next pane, verify that HTTP: is set to Automatic and click Next.
  6. In the Request Handlers list, check ASP.NET, and then check Enable WebDAV publishing and each instance of n:\WINDOWS\Microsoft.NET\Framework\<version number>\aspnet_isapi.dll.
  7. Click Next and then click Finish to complete the wizard.

  • Visual Studio .NET Academic, Professional, Enterprise, or Enterprise Architect
  • Microsoft Message Queuing (MSMQ). For more information about enabling MSMQ, see Section 2.4, Installing Microsoft Message Queuing (MSMQ), in this readme.
  • SQL Server 2000 or Microsoft Data Engine (MSDE)

For increased security, it is highly recommended that the following software be installed prior to installing Assignment Manager server. This software, however, is not required in order for you to use Assignment Manager server.

  • Simple Mail Transfer Protocol (SMTP)

    This service must be running before Assignment Manager is installed if you want to use SMTP for mail delivery.

  • Secure Sockets Layer (SSL)

    You must obtain a secure certificate before installing Assignment Manager if you want to use SSL to protect information that is transferred to and from the Assignment Manager server. For more information, see Secure Sockets Layer: Protect Your E-Commerce Web Site with SSL and Digital Certificates.

To install Assignment Manager, you either need a SQL 2000 server on the local machine to which you are installing or you need to specify a SQL 2000 server on another machine. To work with Assignment Manager, your SQL server must be installed with mixed mode authentication. For more information, see Section 4.8 of this readme, "Must Specify Mixed Mode when Installing SQL Server."

To use a SQL database, you must have the password for the "sa" user of that database. If you do not have this password, then, as a minimum, you need to have the administrator of the database create a new user with the following privileges:

  • Server Roles

    Security Administrators, Setup Administrators, Database Creators, Bulk Insert Administrators

  • Database Access Privileges

    Master (public level)

When installing Assignment Manager, specify the name of the SQL 2000 Server in Step 6 and the user account in Step 8 of the following procedure "To install Assignment Manager".

You can also use the Microsoft Data Engine (MSDE) if you do not wish to install or use an existing SQL 2000 server for Assignment Manager.

To install MSDE

  1. Add the following line to the setup.ini file located at <root directory>\Program Files\Microsoft Visual Studio .NET\Setup\MSDE:
    SECURITYMODE=SQL
  2. Save setup.ini.
  3. From the Start menu, choose Run.

    The Run dialog box appears.

  4. In the Open text box, type the following path:

    <root directory>\Program Files\Microsoft Visual Studio .NET\Setup\MSDE\Setup.exe.

    The Microsoft SQL Server Desktop Engine installer appears. If you want to abort the MSDE installation, click Cancel.

  5. After the MSDE setup completes, reboot your server.

    This action starts the MSSQL$VSdotNET service and the SQL Server Service Manager.

  6. Double-click the SQL Server Service Manager icon in the system tray. When you pass your mouse over this icon, you will see "Not Connected //".

    The SQL Server Service Manager dialog box appears.

  7. Enter <hostname>\VSdotNET in the Server textbox where <hostname> is the machine name for the server on which you are installing MSDE and VSdotNET is the default instance name for the MSDE database. The instance name is specified in the setup.ini file.

    Use the following procedure to determine the machine name.

    1. Right-click My Computer on the desktop and choose Properties.
    2. Click Network Identification and click Properties.

      The machine name is shown in the Computer Name text box.

      Note   If you want to create more than one MSDE database for your Assignment Manager server (for example, one MSDE database per instructor), you can install multiple MSDE instances by editing the instance name in setup.ini and running the MSDE setup for each new instance name.
  8. Click Refresh Services.

    The status bar located at the bottom of the SQL Server Service Manager dialog box will indicate that the service is running. Using this dialog box, you can also stop, pause, or restart a service.

After you have installed MSDE, you need to change the password for the "sa" user from blank to a password of your choosing using the following procedure. The password for the "sa" user must be changed prior to installing Assignment Manager.

To change the default "sa" password for MSDE

  1. From the Start menu, choose Run.

    The Run dialog box appears.

  2. In the Open text box, type cmd.exe.
  3. Browse to the drive and directory where MSDE is installed. By default this is <root directory>\Program Files\Microsoft SQL Server\.
  4. At the command prompt, type the following, and press Enter:
    CD 80\Tools\Binn
  5. At the command prompt, type the following, and press Enter:
    osql -S <hostname>\VSdotNET -U sa

    where <hostname> is the machine name for the server on which you are installing MSDE and VSdotNET is the default instance name for the MSDE database. The instance name is specified in the setup.ini file located at <root directory>\Program Files\Microsoft Visual Studio .NET\Setup\MSDE.

  6. At the Password prompt, press Enter because the default "sa" password installed with MSDE is blank.
  7. At the 1> prompt, type the following and press Enter:
    EXEC sp_password NULL, '<newpassword>', 'sa'

    where <newpassword> is the password of your choosing for the "sa" user.

    Note   The single apostrophes (') around <newpassword> and sa are required.
  8. At the prompt, type the following, and press Enter:
    go

    This executes the password change command.

  9. At the prompt, type the following, and press Enter:
    quit
  10. Type Exit, and press Enter to close the command window.

Following this, install Assignment Manager using the steps provided below. When prompted for the Server Name, Database Name, and User ID, enter values of <hostname>\VSdotNET, AMDB, and sa, respectively.

To install Assignment Manager

  1. Close all open applications.
  2. Set disk quotas on the partition where you are installing Assignment Manager, if appropriate. For more information, see Section 4.13.1 in this readme, "Implementing Disk Quotas for NTFS Partitions."
  3. Insert the CD labeled Visual Studio .NET Academic Teaching Tools.
  4. Open the AssignmentManager folder, and double-click mainsetup.msi to run the Assignment Manager installer.
  5. On the EULA page, read the end user license agreement (EULA). If you accept the terms of the agreement, select I Agree, and click Next to continue with the installation.
  6. Under Server Name, specify the name of the SQL 2000 server. If you are using MSDE for your SQL Server, enter a value of "<hostname>\VSdotNET", where <hostname> is the machine name for the server on which you are installing MSDE and VSdotNET is the default instance name for the MSDE database. The instance name is specified in the setup.ini file located at <root directory>\Program Files\Microsoft Visual Studio .NET\Setup\MSDE. If you have changed the default instance name in the setup.ini file, then enter that name in place of VSdotNET.
  7. Under Database Name, enter the name of your Assignment Manager database. This value is the name of the database created in SQL server that is used to store Assignment Manager data. The default name is "AMDB".
  8. Under User ID, enter a user ID that has administrator-level privileges in the SQL server used for Assignment Manager. This user ID is used to create the SQL database and is only used on installation. A separate SQL user ID and password are created for Assignment Manager. The default value is "sa".
  9. Under Password, enter the password for the user ID that has administrator-level privileges in the SQL server used for Assignment Manager. Passwords are case sensitive.
  10. Click Next.
  11. Clear or select Enable SMTP Messaging and/or Use SSL (HTTPS) for all requests and click Next. These values are selected by default.

    Secure Sockets Layer (SSL) and Simple Mail Transfer Protocol (SMTP) are recommended for use with Assignment Manager. If SSL is not used, then communication between client and server computers is not encrypted and may be compromised. If SMTP is not used, new user passwords for Assignment Manager are set to the user name. If SMTP is used, random passwords are generated by Assignment Manager and e-mailed to new users.

    Note   You cannot enable SMTP or SSL after Assignment Manager is installed.
  12. Under Virtual Directory, enter the virtual directory you will use to create courses. By default, a virtual directory of "AMWeb" is created.
  13. Under Port, a value of "80" is entered by default. Do not change this value. Assignment Manager will ignore other values and continue to use a value of Port 80.
  14. Click Disk Cost to show the list of partitions available for installing Assignment Manager and the required disk space.
  15. Click Next.
  16. Click Next again to begin the installation.
  17. After the installation is complete, click Next.
  18. Click Close to exit the installation.

There is a default user created when you install Assignment Manager server. The user name is "amsa" and the password is "pass". The first time you log on to Assignment Manager, you will be required to change the password for the amsa user. The first person to log on to the Assignment Manager server should create accounts for all administrative and faculty users. To log on for the first time, create a course and enter http://<hostname>/AMWeb for the location of the Assignment Manager server or https://<hostname>/AMWeb if you are using SSL, where <hostname> is the machine name for the server you installed to.

4. Known Issues

4.1. SSL and SMTP Dependencies

Secure Sockets Layer (SSL) and Simple Mail Transfer Protocol (SMTP) are recommended for use with Assignment Manager. If SSL is not used, then communication between client and server computers is not encrypted and may be compromised. If SMTP is not used, new user passwords for Assignment Manager are set to the user name. If SMTP is used, random passwords are generated by Assignment Manager and e-mailed to new users.

4.2. Deleting Assignments and Course Users

Assignments and course users cannot be deleted through the Assignment Manager user interface. They must be deleted through the Assignment Manager SQL server.

Use the following procedure to connect you to Assignment Manager SQL server.

To load and connect to the server

  1. Load Visual Studio .NET.
  2. From the View menu, choose Server Explorer.

    -or-

    Press CTRL+ALT+S to show Server Explorer.

  3. In Server Explorer, right-click Data Connections, and choose Add Connection.

    The Data Link Properties dialog box appears with the Connection tab active.

  4. In the dialog box, enter the SQL server name, log on information, and the database name for your Assignment Manager database. Press OK to connect to the server.

    The database name is the name that you assigned your Assignment Manager database during installation. The default database name is "AMDB".

  5. A connection for the Assignment Manager database is displayed in Server Explorer. Expand this connection to view the SQL tables associated with this database.

Once you have connected to the Assignment Manager SQL server, you can delete a user or an assignment. The Assignment Manager user interface does not currently support either of these actions.

Use the following procedure to delete a course user.

To delete a course user

  • Delete the user record from the following tables in this order:
    1. UserMessages
    2. UserRoles
    3. CourseOfferingUsers
    4. UserAssignments
    5. Users

    In these tables, the UserID field contains the ID of the user you want to delete.

Use the following procedure to delete an assignment from a course.

To delete a course assignment

  • Delete the assignment record from the following tables in this order:
    1. UserAssignments
    2. AssignmentFiles
    3. CourseOfferingAssignments
    4. AssignmentID

    In these tables, the AssignmentID field contains the ID of the assignment you want to delete.

    Note   You can also leave the assignment in the database, but prevent it from showing in the UI, by setting the Status field in the Assignments table to zero (0).

4.12. Add Existing Assignment Feature

The Add Existing Assignment feature makes it possible for an instructor to add an assignment from another course on the same Assignment Manager server to his or her course. The course containing the assignment can belong to that instructor or to another instructor.

The Adding Existing Assignment feature displays an error when the following conditions are true:

  • The original assignment included a starter project.
  • The instructor attempting to add the assignment is not a user of the original course.

To work around this, the instructor who wants to add the existing assignment must be a faculty user of the course containing the original assignment. If the original assignment did not contain a starter project, then this feature does not fail. For more information about adding a user to a course and assigning permissions, see "Assigning User Roles" in the Visual Studio .NET Academic Teaching Tools Guide.

4.13.1. Implementing Disk Quotas for NTFS Partitions

To prevent the drive capacity from being exceeded with output from a single student assignment, use of NTFS is recommended for the drive on which Assignment Manager is installed. The Assignment Manager program creates a user called AssignmentManager that performs auto build and auto check functions. NTFS allows you to set a disk quota for the AssignmentManager user, and thus prevents the drive capacity from being exceeded beyond the quota value you set. Disk quotas cannot be set on FAT or FAT32 partitions. For more information about security problems associated with the FAT and FAT32 file systems, see Section 4.13.3 in this readme, "Potential Security Problems Associated with Auto Build and Auto Check."

Auto build and auto check are performed on one assignment at a time and the output from these tools is deleted from the disk before the tools are run on the next assignment. As a result, the maximum disk quota allotted to the AssignmentManager user should be determined from the size of the NTFS partition to which the quota is applied and the expected auto build and auto check output for a single student assignment.

Use the following procedure to enable disk quotas on a Windows 2000 Server, Windows 2000 Advanced Server, Windows .NET Standard Server, or Windows .NET Enterprise Server. Disk quotas should be enabled prior to installing Assignment Manager.

To enable disk quotas for your Assignment Manager server

  1. Double-click My Computer on the desktop.
  2. Right-click the icon for the NTFS partition for which you want to enable disk quotas, and choose Properties.
  3. Click the Quota tab, and select the Enable quota management check box.
  4. Select the options you want to apply to your NTFS partition.
    Note   Disk quotas are disabled by default for Windows 2000 Server, Windows 2000 Advanced Server, Windows .NET Standard Server, and Windows .NET Enterprise Server.

After enabling disk quotas, install Assignment Manager. For more information, see Section 2.3 in this readme, "Installing Assignment Manager."

After you have installed Assignment Manager with auto check and auto build enabled, you need to activate the AssignmentManager user before you can set disk quotas for this user.

To activate the AssignmentManager user

  1. Create an assignment that uses auto build, and auto check in Assignment Manager.
  2. Add this course to the student client, and then submit this assignment to Assignment Manager through the student client.

    When Assignment Manager builds and checks this assignment using the auto build and auto check features, respectively, the AssignmentManager user is activated and an entry for this user is placed in the quota entries table.

For more information about these steps, see the Visual Studio .NET Academic Teaching Tools Guide and the Visual Studio .NET Academic Student Tools Guide.

You can now set the disk quota for this user using the following procedure.

To set disk quotas for the AssignmentManager user

  1. Double-click My Computer on the desktop.
  2. Right-click the icon for the NTFS partition with enabled disk quotas, and choose Properties.
  3. Click Quota Entries.

    The AssignmentManager user name is displayed in the quota entries table.

  4. Double-click AssignmentManager from the list of quota entries.
  5. On the General tab, set quotas for the AssignmentManager user.

As an alternate approach, you can enable disk quotas for all new users of the server prior to installing Assignment Manager using the following procedure. This approach applies one quota to all new users. This quota will be applied to the AssignmentManager user, the first time that auto build and auto check are run.

To enable disk quotas for all new users

  1. Double-click My Computer on the desktop.
  2. Right-click the icon for the NTFS partition for which you want to enable disk quotas, and choose Properties.
  3. Click the Quota tab, and select the Enable quota management check box.
  4. Click Limit disk space to, and enter a limit in the text box.
    Note   This limit is the default limit applied to all new users of this NTFS partition. You can edit the limit for each new user using the procedure specified above for setting the disk quota for the AssignmentManager user. Existing users of the NTFS partition retain their individual quotas.

4.13.2. FAT and FAT32 Security Issues

Members of the Users group on a Windows 2000 Server or Windows 2000 Advanced Server have read/write access on FAT and FAT32 disk partitions. The AssignmentManager user, which performs auto build and auto check functions, is a member of this group. By extension, student programs that are executed by the auto check feature have read/write access to the partition. Installing Assignment Manager on an NTFS partition is a more secure solution than using a FAT or FAT32 partition because the Users group does not have access to write to the system folders, and because you can set disk quotas for the user.

If Assignment Manager is installed on a FAT or FAT32 partition, student users can read and write files at any location on the partition. This can be a security issue for a number of reasons. For example, the student could read secure information or could write files that are automatically run and can potentially cause damage to the system.

It is recommended that you use NTFS for the partition. NTFS supports file permissions, while FAT and FAT32 do not.

4.13.3. Potential Security Problems Associated with Auto Build and Auto Check

Auto build and auto check allow untrusted code to be compiled and run on the Assignment Manager server. If the auto build and auto check features of Assignment Manager are enabled, then the student's code represents a potential threat to security. Specifically, students may be able to gather information about the server or cause damage to the server through the code they submit.

If you are concerned about security risks, you can disable auto build and auto check. For more information about how to disable auto build and auto check, see "Modifying Server Settings" in the Visual Studio .NET Academic Teaching Tools Guide.

4.15 Adding Additional Roles to the Administrator

Users with the Administrator role should not add additional roles for themselves, such as the Faculty role. If a user with the Administrator role does try to add another role to themselves, an error message is displayed and the user loses all privileges.

To work around this, add a new user with administrator privileges, and then log in as the new Administrator. After you have logged in as the new Administrator, edit the roles of the original Administrator. For more information, see "Default Assignment Manager Role Permissions and Accessing User Accounts" in the Visual Studio .NET Academic Teaching Tools Guide.

4.16 Including Extra Files with Assignments

Assignments that include extra files must also include a starter project, otherwise students cannot see the extra files associated with the assignment when they view it. For information on creating a starter project, see "Adding an Assignment" in the Visual Studio .NET Academic Teaching Tools Guide.

4.17 Double-byte Characters do not Display Correctly in Chinese, Japanese, and Korean Languages

Identify all file paths and course names by using only single-byte characters. Double-byte characters might display as a square.

4.18 Restricted Characters for Assignment Names

Assignment names can use only the characters a-z, A-Z, and 0-9. If you use characters other than those listed, such as double-byte characters, you run the risk of duplicate assignment names.

4.19 Debugging Applications on Terminal Server Machines Running Visual Studio

Students who use Visual Studio from a Terminal Server machine do not, by default, have the ability to debug their applications from Terminal Server. Students must be added to the Debugger Users Group before they can debug their applications.

Note   Assignment Manager should not be installed on the same machine on which students are given access to debug via Terminal Server, because anyone added to the Debugger Users Group can stop and debug any application running from that machine.

To enable students to debug on a machine with Terminal Server

  1. For Windows 2000, on the Start menu, choose Settings and then choose Control Panel. For Windows XP or later, on the Start Menu, choose Control Panel.
  2. For Windows 2000, choose Users and Passwords. For Windows XP or later, Choose User Accounts.
  3. In the Users for this computer list, select the users you intend to add to the Debugger Users Group and click Properties.
  4. On the Group Membership tab, click Other and then choose Debugger Users.
  5. Click OK and then click OK again.

To enable domain users to debug on a machine with Terminal Server

  1. For Windows 2000, on the Start menu, choose Settings and then choose Control Panel. For Windows XP or later, on the Start Menu, choose Control Panel.
  2. For Windows 2000, choose Users and Passwords. For Windows XP or later, Choose User Accounts.
  3. Click Add.
  4. Type the user name and the domain and then click Next.
  5. Choose Other and then choose Debugger Users.
  6. Click OK and then click OK again.
