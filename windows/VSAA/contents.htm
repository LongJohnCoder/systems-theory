2. Installation

Read the following sections for information on installing the Visual Studio .NET Academic Assignment Manager Source Package.

2.1. Visual Studio .NET Academic Assignment Manager Source Package Compatibility

Visual Studio .NET Academic Assignment Manager Source Package cannot be modified in Visual Studio .NET 2002, nor can modified versions of Visual Studio .NET Academic Assignment Manager be accessed by machines with Visual Studio .NET 2002 installed. Assignment Manager Source is compatible only with Visual Studio .NET 2003.

2.2. Installing Microsoft Visual Studio .NET Academic Assignment Manager Source Package

Use the following procedure to install Visual Studio .NET Academic Assignment Manager Source Package.

To install Assignment Manager Source Package

  1. Double-click AssignmentManagerSource.exe.

    A message box appears, with information about the source package.

  2. In the message box, click OK.
  3. In the next dialog, choose the location where you want the files copied to or accept the default location and click Unzip.

    When the files have been copied, the readme file is displayed.

Next, create a virtual directory for the Web application portion of Assignment Manager. To use and customize Assignment Manager, Internet Information Services (IIS) must be installed.

To create a virtual directory for Assignment Manager

  1. On the Start menu, choose Run.
  2. Enter inetmgr and then click OK.
  3. In the Internet Information Services (IIS) Manager, expand <machine name> (local computer) and then expand Web Sites.
  4. Right-click Default Web Site, select New, and then select Virtual Directory.

    The Virtual Directory Creation Wizard appears.

  5. Click Next.
  6. In the Alias text box, type AMWebDev and then click Next.
    Note   Assignment Manager Source Package cannot be installed on a machine that already has a Web application that uses the alias "AMWebDev".
  7. In the Directory box, enter the path to the location where \...\Assignment Manager Server\Web\AMWebDev\ was installed, or click Browse to locate the path and then click Next. By default, this location is c:\AssignmentManagerSource\Assignment Manager Server\Web\AMWeb.
  8. Choose the level of access permissions for the virtual directory and then click Next.
  9. Click Finish.

Next, install the Assignment Manager deployment dialog.

To install the deployment dialog

  1. In the installation directory for the source package, copy the file VsdCustomTextPwd.wid. By default, this location is c:\AssignmentManagerSource\Assignment Manager Server\setup\dialogs\VsdCustomTextPwd.wid.
  2. Browse to \...\Microsoft Visual Studio .NET 2003\Common7\Tools\Deployment\VsdDialogs\1033 and paste VsdCustomTextPwd.wid in the directory. This path depends on where you chose to install Visual Studio .NET 2003.

If you do not install the deployment dialog, the following error message appears when you attempt to build the Assignment Manager deployment project in Visual Studio:

"ERROR: Unable to build dialog module named 'C:\Program Files\Microsoft Visual Studio .NET 2003\Common7\Tools\Deployment\.\VsdDialogs\1033\VsdCustomTextPwd.wid'"

4. Distributing Modified Versions of Academic Assignment Manager

You can distribute the source code for Academic Assignment Manager using the Shared Source License agreement for Visual Studio .NET Academic Assignment Manager Source Package. This license agreement can be found in the root folder where you extracted the source package from. By default, this location is c:\assignmentmanager.

If you choose to distribute a custom version of Assignment Manager, Assignment Manager Faculty Client, or Assignment Manager Student Client, your modified version must include a license agreement. The license agreement you provide should exist as an .RTF file and included in the deployment project for the binary files you wish to distribute.

To include a license agreement in your deployment projects

  1. In Solution Explorer, right-click on the deployment project you want to add the license to, select View, and then select User Interface.

    For Assignment Manager, the deployment project is AMSetup; for Assignment Manager Faculty Client the deployment project is FacultyClientSetup: and for Assignment Manager Student Client the deployment project is StudentClientSetup.

  2. In the User Interface editor, select License Agreement, found by expanding Install and then expanding Start.
  3. In the Properties window, select License File and then select Browse.
  4. In the Select Item in Project dialog box, navigate to the .RTF file that contains your license agreement, and then click OK.
  5. On the Build menu, choose Build <projectname>, where projectname is the name of the deployment project you just modified.

The license agreement is now included in your .msi file.
