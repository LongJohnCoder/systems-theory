2. Installation

Visual Studio .NET Academic is provided with the following six CDs:

  • Windows Component Update
  • Visual Studio .NET Academic CD1
  • Visual Studio .NET Academic CD2
  • Visual Studio .NET Academic CD3
  • Visual Studio .NET Academic CD4
  • Visual Studio .NET Academic Student Tools
Note   Install Visual Studio .NET Academic before you install the Student Tools.

2.1. Installing Microsoft Visual Studio .NET Academic Student Tools

Use the following procedure to install Visual Studio .NET Academic.

To install the Academic Student Tools

  1. Install Visual Studio .NET Academic. For more information, see the Readme.htm file located at the root directory of the Visual Studio .NET Academic CD1.
  2. Insert the CD labeled Visual Studio .NET Academic Student Tools into your CD-ROM drive.
  3. Run Setup.msi from the root directory of the Visual Studio .NET Academic Student Tools CD-ROM.
  4. Read and accept the end user license agreement (EULA).
  5. After you accept the EULA, the Student Tools are installed on the drive that contains Visual Studio.

2.2. Uninstalling Previous Versions of Visual Studio .NET Academic Student Tools

Use the following procedure to uninstall Visual Studio .NET Academic.

Caution   If you have a previously installed version of Visual Studio .NET Academic on your computer, you must complete the following steps to uninstall the previous release. Installing Visual Studio .NET Academic on a machine where previous versions of the .NET Framework or Visual Studio .NET have been installed and uninstalled is not supported.

To uninstall previous versions of Visual Studio .NET Academic

  1. From the Start menu, point to Settings, and then click Control Panel.

    The Control Panel opens.

  2. Click Add/Remove Programs, and then click Change or Remove Programs.
  3. If present, select Visual Studio .NET, and click Change/Remove. Otherwise, skip to step 7.
  4. Click link 2, Visual Studio .NET.
  5. Click Uninstall Visual Studio .NET to begin uninstalling.
  6. After the previous version of Visual Studio .NET has been uninstalled, return to the Add/Remove Programs window.
  7. Select Microsoft .NET Framework, and choose Change/Remove.
  8. Answer Yes to proceed with removal.

To uninstall Student Tools Beta 2

  1. From the Start menu, point to Settings, and then click Control Panel.

    The Control Panel opens.

  2. Double-click Add/Remove Programs.
  3. Select Change or Remove Programs in the left pane of the Add/Remove Programs dialog box.
  4. Select Academic Student Tools - Language.
  5. Click Remove.

4. Known Issues

This section provides a listing of known issues associated with Visual Studio .NET Academic Student Tools.

4.1. Accessing Assignment Manager from Outside of Visual Studio .NET Academic

While it is possible to access the Assignment Manager user interface from outside of Visual Studio .NET Academic, such access is not supported. Specifically, the uploading and downloading of files by student or faculty users does not work when Assignment Manager is accessed through a Web browser rather than Visual Studio .NET.

4.2. Project Types Supported by Assignment Manager

Assignment Manager only supports use of Visual C#, Visual Basic, and Visual C++ project types for starter projects uploaded with assignments and for student project submissions.

Note   Projects that are stored using source code control cannot be uploaded to the Assignment Manager server. You can add the project to source code control after it has been uploaded to Assignment Manager.

The following table shows the Visual Basic and Visual C# project types that are supported.

Template Supported in auto build Supported in auto check
Windows Application Yes No
Class Library Yes No
Windows Control Library Yes No
Console Application Yes Yes
Windows Service Yes No
Empty Project Yes Yes

The following table shows the Visual C++ project types that are supported.

Template Supported in auto build Supported in auto check
Advanced Placement Console Application Yes Yes
C/C++ Console Application Yes Yes
Managed C++ Application Yes Yes
Managed C++ Class Library Yes No
Win32 Application Wizard (Console Application) Yes Yes
Win32 Application Wizard (Windows Application) Yes No
DLL Yes No
Static Library (non-MFC) Yes No

The following project types are not supported:

  • Web projects
  • Setup and Deployment projects
  • Database projects
  • Enterprise Template projects
  • Visual Studio Analyzer projects
  • Extensibility projects
  • Application Test Center projects
  • ATL projects
  • Custom Wizard projects
  • Makefile projects
  • MFC Application projects

4.3. Loading Pages in Assignment Manager

The first time a new Assignment Manager installation is used by the first student user to access it, a runtime error may occur when the student user moves the mouse over a page that has not finished loading.

Resolution: When this error occurs, click Refresh and the page will load correctly.

4.4. Cannot Upload Projects from a Share

In Assignment Manager, you can only upload a project from a local folder on the machine that you are currently using. If you attempt to upload a project as part of your completed assignment and the project is located on another machine, the upload will fail and you will receive the following error message: Starter project could not be uploaded to the Assignment Manager server. Either the student code extraction failed or the project type is not supported.
