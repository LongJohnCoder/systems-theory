SEM234: Operating Systems

Eike Ritter

10 credits in Semester 2, year 2

Aims

  • Explain the role and function of an operating system
  • Show how resource management is done in an OS
  • Examine the additional complexities of distributed OS
  • Evaluate security mechanisms in OS

Objectives

On completion of this course, the student should be able to:
  • Understand the role of an operating system
  • Understand how the decomposition of a program into processes affects its behaviour.
  • Analyse known scheduling algorithms and adapt them for new situations.
  • Suggest appropriate memory management algorithms for a variety of situations.
  • Find the additional problems when transferring a concrete situation from a monoprocessor system to a distributed environment and solve the arising co-ordination problems.
  • Devise appropriate security mechanisms in a given situation and understand the role operating system can and should play in establishing security.

Prerequisites

Basic knowledge of Computer Architecture (e.g. SEM109) and basics of algorithms (SEM117).

Teaching Methods

20 hours of lecture together with exercises classes. I will give out an exercise sheet every two weeks which I will mark on a voluntary basis and provide a class test during reading week, which the students will mark themselves. In addition there will be practical exercises, which will consist in implementing some of the algorithms described in the course. Neither the exercise sheets nor the class test nor the practical exercises count towards the module mark.

Assessment

Two hour unseen exam in May.

Detailed Syllabus

  1. Function of Operating Systems
    I start the course by explaining the function of an operating system in general terms and list necessary prerequisites of the hardware. (2 lectures)
  2. Processes
    The concept of a process is fundamental for an operating system. I will discuss the states of a process and discuss process scheduling in some detail because it is paradigmatic for many other resource allocation progblems in operating systems. Afterwards I will discuss threads (lightweight processes). They are introduced to reduce the time necessary for context switches. (4 lectures)
  3. Memory Management
    An absolutely crucial aspect of every operating system is memory management. It arises because main memory is a limited resource, which has to be used as efficiently as possible. This part discusses memory management strategies and their dependence on the behaviour of programs on one side and on properties of the hardware on the other side. I will discuss virtual memory in detail. (4 lectures)
  4. File Systems
    Again, the physical constraints of a disk have a great impact on the design of a file system. This part of the course shows how the principles discusses for process scheduling and memory management apply to file systems. (3 lectures)
  5. Distributed Systems
    A whole series of new problems arise when one considers distributed operating systems. I will present the various models that have been proposed (shared memory, loosely coupled etc.) and discuss their relative merits. I will stress co-ordination problems which are at the heart of distributed systems. (4 lectures)
  6. Security
    I will discuss various meanings of the term ``security'': data integrity, protection against user errors and protection against malicious users. I will use access control as an example to illustrate common problems in this area. (3 lectures)
Synchronisation is an important part of any operating system. This is not part of this course because it is covered in detail in SEM227 Real-time systems programming. Also I will omit material about networks which is a separate course.
